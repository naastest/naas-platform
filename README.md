# NaaS Platform — Local Dev Guide

Namespace-as-a-Service demo on a single Minikube cluster. Teams request namespaces via PRs; everything downstream (quotas, network policies, RBAC, ArgoCD scoping) is generated automatically from namespace labels.

## What runs here

| Component | URL | Purpose |
|-----------|-----|---------|
| **ArgoCD** | https://argocd.naas.local | GitOps engine — syncs this repo |
| **Authentik** | https://authentik.naas.local | OIDC provider for ArgoCD + Kubernetes |
| **Gangplank** | https://gangplank.naas.local | Self-service kubeconfig portal |
| **capsule-proxy** | https://capsule-proxy.naas.local | Multi-tenant API gateway (namespace filtering) |

## Prerequisites

```bash
brew install minikube skaffold kubectl helm argocd kyverno
brew install --cask docker
```

Also install [kubectl-oidc-login](https://github.com/int128/kubelogin#setup):
```bash
brew install int128/kubelogin/kubelogin
```

## First-time setup

### 1. Start the cluster

```bash
make up
```

This runs `./bootstrap/00-minikube-start.sh` (starts Minikube with OIDC flags) then `skaffold run -p local` to install:

- sealed-secrets, cert-manager, ingress-nginx, Kyverno, Capsule, ArgoCD, Authentik, capsule-proxy

Takes 5–10 minutes on first run (image pulls).

### 2. Open a tunnel — required for browser/OIDC access

In a **separate terminal** (keep it open):

```bash
make tunnel
# or: minikube tunnel --profile naas-local
```

This exposes the ingress-nginx LoadBalancer on `127.0.0.1:443`. Without it, `*.naas.local` URLs are unreachable from your Mac.

### 3. Fix DNS and TLS

```bash
make post-start
```

This runs three scripts:

- **`make hosts`** — adds `*.naas.local` entries to `/etc/hosts` pointing to `127.0.0.1`
- **`make fix-ca`** — injects the cert-manager CA into Minikube's trust store (required for kube-apiserver OIDC TLS) and saves it to `~/.kube/naas-local-ca.crt`
- **`make coredns-patch`** — patches CoreDNS so pods can resolve `*.naas.local` to ingress-nginx

### 4. Hand over to ArgoCD

```bash
make bootstrap
```

Applies `platform/app-of-apps.yaml`. ArgoCD then manages all remaining platform config (Authentik blueprints, Kyverno policies, ingresses, AppProjects, namespace provisioning).

### 5. Configure Authentik (one-time, after Authentik is ready)

1. Open https://authentik.naas.local
2. Log in with the bootstrap credentials from `bootstrap/06-authentik/values.yaml`
3. ArgoCD has already synced the `platform-authentik-config` app which pushes the blueprint ConfigMap into the `authentik` namespace
4. Authentik's blueprint runner auto-applies `/blueprints/custom/*.yaml` — wait ~60 s for it to complete
5. Check **Admin → System → Blueprints** — all three providers (Kubernetes, ArgoCD, Gangplank) and the platform groups should show `Successful`

## Daily workflow

### Starting after a machine restart

```bash
minikube start --profile naas-local   # restores the existing cluster
make post-start                        # re-runs hosts/CA/CoreDNS fixups
# In a separate terminal:
make tunnel
```

`make post-start` is needed after every Minikube restart because:
- The CA cert in the Minikube container is ephemeral (lost on stop/start)
- The CoreDNS ConfigMap is reset when the cluster restarts

### Stopping

```bash
minikube stop --profile naas-local    # suspends cluster, keeps state
# Ctrl-C the tunnel terminal
```

### Full reset (wipe and rebuild)

```bash
make reset
# Then repeat the first-time setup steps 2–5
```

## Getting a kubeconfig via Gangplank

Gangplank is the self-service portal for generating kubeconfigs that point at capsule-proxy (which filters namespaces by team membership).

1. Open https://gangplank.naas.local
2. Click **Login** — you'll be redirected to Authentik
3. After login, go to **Command line** (https://gangplank.naas.local/commandline)
4. Copy the `kubectl config` commands and run them in your terminal
5. Switch to the new context and test:

```bash
kubectl get namespaces   # shows only namespaces your groups can access
```

The kubeconfig uses capsule-proxy as the API server. Namespace filtering is based on RoleBindings generated by Kyverno from namespace labels.

## Troubleshooting

### `*.naas.local` not reachable in browser

**First check:** is the tunnel running? Start it in a separate terminal:

```bash
make tunnel
```

**If the tunnel crashes immediately with TLS errors** (`x509: "minikube" certificate is not standards compliant`):

`minikube update-context` sometimes resets the `naas-local` kubeconfig user to empty `{}`, breaking TLS auth. Fix it:

```bash
make fix-kubeconfig   # restores client cert credentials in ~/.kube/config
make tunnel           # now works
```

`make post-start` includes `fix-kubeconfig` automatically.

If the tunnel is running but URLs still fail:

```bash
# Check the LoadBalancer got an IP
kubectl get svc -n ingress-nginx ingress-nginx-controller
# EXTERNAL-IP should be 127.0.0.1, not <pending>

# Re-run hosts fixup
make hosts
```

### `*.naas.local` not reachable from inside the cluster (pod-to-pod)

CoreDNS patch is missing or stale.

```bash
make coredns-patch
```

### Authentik OIDC errors (ArgoCD login / kubelogin / Gangplank)

**"invalid bearer token"** or **"cannot authenticate the token"**:
- Old cached token. Delete `~/.kube/naas-token-cache/` and re-login.
- Or re-generate the kubeconfig via Gangplank.

**ArgoCD shows OIDC error after cluster restart**:

```bash
# Restart ArgoCD server (picks up fresh OIDC discovery)
kubectl rollout restart deployment/argocd-server -n argocd
```

**Authentik blueprints stuck / not applied**:

```bash
# Check blueprint status
kubectl exec -n authentik deploy/authentik-worker -- ak shell -c "
from authentik.blueprints.models import BlueprintInstance
for b in BlueprintInstance.objects.all():
    print(b.name, b.status)
"

# Force re-run all blueprints
kubectl exec -n authentik deploy/authentik-worker -- ak shell -c "
from authentik.blueprints.models import BlueprintInstance
from authentik.blueprints.tasks import apply_blueprint
for b in BlueprintInstance.objects.all():
    apply_blueprint.delay(str(b.pk))
    print('Queued:', b.name)
"
```

### kube-apiserver OIDC TLS fails after cluster restart

The CA cert is lost when Minikube stops. Run:

```bash
make fix-ca
# Then restart kube-apiserver to pick up the new CA
kubectl delete pod -n kube-system -l component=kube-apiserver
```

### Kyverno policies blocking namespace creation

```bash
# Check policy violations
kubectl get policyreport -A
kubectl describe policyreport -n <namespace>
```

### ArgoCD app stuck OutOfSync

```bash
make status                     # overview
argocd app sync <app-name>      # manual sync
argocd app diff <app-name>      # see what's diverged
```

## Architecture overview

```
┌─── Your Mac ────────────────────────────────────────────┐
│                                                          │
│  Browser / kubectl                                       │
│       │                                                  │
│       │ 127.0.0.1:443 (minikube tunnel)                 │
│       ▼                                                  │
│  ┌─── Minikube (naas-local) ─────────────────────────┐  │
│  │  ingress-nginx                                      │  │
│  │       │                                             │  │
│  │  ┌────┴────────────────────────────────────────┐   │  │
│  │  │  argocd.naas.local → ArgoCD                 │   │  │
│  │  │  authentik.naas.local → Authentik (OIDC)    │   │  │
│  │  │  gangplank.naas.local → Gangplank (portal)  │   │  │
│  │  │  capsule-proxy.naas.local → capsule-proxy   │   │  │
│  │  └─────────────────────────────────────────────┘   │  │
│  │                                                     │  │
│  │  ArgoCD manages:                                    │  │
│  │    platform/app-of-apps.yaml                        │  │
│  │    └── namespaces/* → Namespace + labels            │  │
│  │         └── Kyverno generates:                      │  │
│  │               ResourceQuota, LimitRange             │  │
│  │               NetworkPolicy, RoleBindings           │  │
│  │    └── tenants/* → Capsule Tenant CRs               │  │
│  │    └── policies/* → Kyverno ClusterPolicies         │  │
│  └─────────────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────────────┘
```

**Auth flow** (Gangplank → capsule-proxy → kube-apiserver):
1. User logs in at `gangplank.naas.local` via Authentik OIDC
2. Gangplank generates a kubeconfig pointing at `capsule-proxy.naas.local`
3. `kubectl` sends the OIDC Bearer token to capsule-proxy
4. capsule-proxy forwards it to kube-apiserver (OIDC validated there)
5. kube-apiserver impersonation headers returned; capsule-proxy filters namespace list by RoleBinding membership

**OIDC config** (important for cluster recreation):
- Issuer: `https://authentik.naas.local/` (root Authentik URL)
- Client ID: `gangplank` (Gangplank Authentik provider, `issuer_mode: global`)
- kube-apiserver flags: `--oidc-issuer-url`, `--oidc-client-id=gangplank`, `--oidc-username-claim=preferred_username`

## Adding a new team namespace

```bash
make ns-add TEAM=myteam TIER=backend ENV=dev APP=my-service COMPLIANCE=standard COST_CENTER=eng-001
# Edit the generated file, then commit and push
# ArgoCD picks it up automatically; Kyverno generates quotas/RBAC/NetworkPolicies
```

## Make targets reference

```
make up              Start Minikube + deploy all platform components
make down            Remove Helm releases (keep Minikube running)
make reset           Full wipe and rebuild
make tunnel          Start Minikube tunnel (keep open in separate terminal)
make post-start      Re-run hosts + CA + CoreDNS fixups (needed after every restart)
make hosts           Write /etc/hosts entries for *.naas.local
make fix-ca          Inject cert-manager CA into Minikube + ~/.kube
make coredns-patch   Patch CoreDNS to resolve *.naas.local inside the cluster
make bootstrap       Apply ArgoCD app-of-apps (hand over GitOps control)
make status          Show ArgoCD app sync status
make ns-add          Scaffold a new namespace directory
make validate-ns     Validate namespace label schemas
make kyverno-test    Run Kyverno policy unit tests
make help            Show all targets
```
